# Agentic AI Usage Rules & Playbook

## 1. Purpose
Ensure consistent, safe, and effective use of Agentic AI tools during the Vibe Coding session so BA, Dev, and QA contributors can collaborate in parallel without conflicting outputs.

## 2. Guiding Principles
1. **Human-in-the-loop**: AI drafts require human review before acceptance.
2. **Single source of truth**: Implementation plan + DAR supersede AI suggestions when conflicts arise.
3. **Prompt specificity**: Provide requirement IDs, file paths, and context to minimize hallucinations.
4. **Traceability**: Log all AI-assisted decisions in documentation (DAR/change log) when they affect scope or architecture.
5. **Security**: Never paste secrets, credentials, or personal data into AI prompts.

## 3. Role-Based AI Usage
| Role | Recommended Prompts | Guardrails |
|------|---------------------|------------|
| BA | "Summarize REQ-00X for scope doc", "Generate acceptance criteria for guest checkout." | Verify alignment with task guide; ensure requirements stay within scope/timebox. |
| Dev | "Produce Express controller skeleton for `/api/orders`", "Draft Zustand store for auth." | Cross-check against implementation plan (Prisma schema, JWT strategy); no direct dependency installs without approval. |
| QA | "Generate test cases for REQ-003 cart handling", "Create Postman assertions for `/api/auth/login`." | Validate negative scenarios manually; ensure tests map to requirement IDs. |
| Doc Engineer | "Rewrite section for clarity", "Create table summarizing decisions." | Maintain consistent formatting/templates; cite source sections (spec vs. plan). |

## 4. Prompt Templates
```
You are assisting with the Agentic AI E-Commerce project.
Context:
- Requirement ID: REQ-003 (Cart Management)
- Stack: React frontend, Express + Prisma backend, SQLite
Task:
[Describe task]
Constraints:
- Follow color palette (red/black/gray/white)
- JWT auth with refresh cookie
Provide output in [format].
```

## 5. Review & Validation Checklist
1. Confirm AI output references correct requirement/decision IDs.
2. Compare code/doc suggestions with prisma schema + API spec.
3. Run lint/tests or manual validation after applying code generated by AI.
4. Capture notable AI decisions (e.g., new helper function) in change log.
5. During demo, attribute AI contributions but highlight human review.

## 6. Escalation Path
- If AI output conflicts with prior decisions → log in DAR/Change Log and resolve with team lead before continuing.
- If AI suggests scope creep → move item to risk/open questions section.
- If tools produce repeated errors → switch to manual drafting to avoid time loss.

## 7. Tooling Inventory
- **Cascade/Windsurf** (this workspace) for code + doc edits.
- **Swagger/OpenAPI** for API referencing.
- **Postman/Newman** optional for QA automation (manual creation if AI assistance insufficient).

## 8. Compliance & Ethics
- Do not feed proprietary third-party content into AI prompts without clearance.
- Maintain workshop confidentiality; outputs stay in repo or approved drive.
- Respect participant roles; AI is augmentative, not a replacement for collaborative review.

## 9. Traceability
- Links REQ-007 and DAR D-09/D-10 (documentation & AI rules) ensuring all downstream artifacts know how AI contributions were governed.
